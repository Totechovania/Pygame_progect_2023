ТЗ:

1) Код:
	-структура:
		Структура игры будет строиться из самых главных основных классов из которых будут собираться игровые объекты, а уже из них сцены, по типу:
		BetterSprite - улучшенный класс для работы со спрайтами который будет использоваться на протяжении всего создания игры
2) Визуализация:
	-меню, кнопки, игровые объекты:
		для создания дизайна для этих объектов непосредственно на момент дизайнинга уже готовой игры, будут просмотреты библиотеки которые можно использовать для улучшения качества дизайна,
		а при их отсутствии прийдется делать дизайн вручную пользуясь возможностями pygame,
		также при визуализации будут применены различные спрайты и картинки из интернета для улучшения внешнего вида игры
3) Звук:
	-кнопки, действия:
		при использовании игровых объектов (взаймодействия с ними) будет проигрываться какой-то звук (он будет выбран позже на стадии реализации) средствами pygame
	-музыка:
		во время игры будет проигрываться музыка (она также будет выбрана позже) и пользователя будет иметь возможность убавить ил прибавить громкость так и выключить ее совсем ( также и со 				внутреигровыми звуками)
4) Геймдизайн:
	-практически все игровые объекты:
		будут либо скачаны (при наличии) либо созданы вручную, также как и в основной игре Antiyoy (разноцветные кнопки не высокой яркости, а также государства разных цветов (для отличения себя от 			противников ( но это уже мелочи )))
5) Хранение данных:
	-прогресса в игре, а возможно и текущей сессии (это будет реализованно при достаточном количестве времени):
		данные будут сохраняться в базу данных sqlitestudio в соответсвующую таблицу
6) Название игры:
	-MyAntiyoy

Основные сцены:

#0) Возможно будет реализована заставка
1) Начальное и по совместительству главное меню:
	-кнопка "Настройки"
		-настройки звука и музыки
		-автосохранение (возможно будет реализовано)
		-язык (возможно будет реализовано)
		-дополнительная информация
	-кнопка "Выход"
		-выходит из игры
	-кнопка "Начать"
		-открывается другое окно в котором будут следующие кнопки:
			-редактор (создание карты вручную или копирование из txt файла) (возможно будет реализовано)
			-сражение (быстрый выбор настроек для быстрой игры)
			-кампания (уже заранее созданые уровни для открытия следующих необходимо их прохожжение в определенной последовательности)
		
2) Всплювающие окна (например во время боя или просмотра доп информации)
3) Сцена перехода между окнами
4) Сцена победы или поражения:
	-на ней будет статистика и появляться она будет после завершения какого либо боя
		- кнопка завершить
		- кнопка заново
5) Сама сцена боя (карта с которой есть возможность взайимодействовать):
	-счетчик монет
	-зарабаток за ход
	-выход в меню
	-вернуть действие назад
	-завершить ход
